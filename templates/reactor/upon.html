{% extends "layout.html" %} {% block markdown %} The *upon* Reactor == This critical reactor manages the triggering of jQuery method calls when events are received on an element. The events and methods can be of any nature, as well as the arguments the methods receive. Data Arguments to *upon* ===
data-upon:source
Specifies the element which will be watched for events. By default it is *this* element, but for cases like the button enable example, watching other elements for things like change can be very useful.
data-upon:target
Specifies wihch element will have method(s) invoked when the source element has received watched events. If not specified, *this* is assumed.
data-upon:event.method
data-upon:event.method#argument...
data-upon:event.method.argument...
Indicates which method or methods to apply to the target. Arguments are specified as structured data or as an array that is applied as arguments. See the section regarding data objects and arguments for more details.
data-upon:event.source
data-upon:event.target
Same as the more global versions but applies to only one event.
data-upon:event.method.source
data-upon:event.method.target
Same as the more global versions but applies to only one method.
The Basics === *upon* manages the automatic binding of events from a *source* to a *target*, each defaulting to *this* if not provided. *upon* then connects various *event* bindings to the *methods* to call, as well as the arguments to apply.
